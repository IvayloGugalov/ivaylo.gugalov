# Portfolio Site — Auth Agent Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire up Better Auth (GitHub OAuth), the `requireAuth` oRPC middleware, the auth API route handler, and the client-side Zustand auth store.

**Architecture:** Better Auth handles all OAuth flow at `/api/auth/*`. A `requireAuth` procedure wrapper guards protected oRPC endpoints. The Zustand store is a thin cache of the server-side session on the client.

**Package manager: bun** — use `bun run <script>` for all scripts. Bun reads `.env.local` automatically.

**Tech Stack:** better-auth, drizzle-orm, zustand, @tanstack/react-router

**Prerequisite:** DB agent complete (schema.ts and db client exist).

### Task 3: requireAuth oRPC middleware

**Files:**
- Create: `src/server/auth/middleware.ts`

**Important:** oRPC middleware uses a single destructured object `{ input, context, next }` — NOT a three-argument signature. The `next()` call merges new context.

- [ ] **Step 1: Write requireAuth procedure wrapper**

Create `src/server/auth/middleware.ts`:

```ts
import { os } from '@orpc/server'
import { auth, type Session, type User } from './auth'

// Base procedure — all procedures start here
export const baseProcedure = os

// Public procedure — no auth required
export const publicProcedure = baseProcedure

// Protected procedure — throws UNAUTHORIZED if no valid session
export const requireAuth = baseProcedure.use(
  async ({ context, next }) => {
    const headers = (context as { headers?: Headers }).headers
    if (!headers) throw new Error('UNAUTHORIZED')

    const session = await auth.api.getSession({ headers })
    if (!session?.user) throw new Error('UNAUTHORIZED')

    return next({
      context: {
        ...(context as object),
        session: session as Session,
        user: session.user as User,
      },
    })
  }
)
```

> **Note for query-orpc-agent:** Import `publicProcedure` and `requireAuth` from this file when building routers.

### Task 5: Auth oRPC router (me + logout)

**Files:**
- Create: `src/server/auth/router.ts`

**Note:** oRPC procedure handlers receive a single destructured object `{ input, context }`. The `context` here includes `headers` injected by the fetch handler.

- [ ] **Step 1: Write auth router**

Create `src/server/auth/router.ts`:

```ts
import { os } from '@orpc/server'
import { auth } from './auth'
import { publicProcedure, requireAuth } from './middleware'

export const authRouter = os.router({
  // Returns current session user or null — public
  me: publicProcedure.handler(async ({ context }) => {
    const headers = (context as { headers?: Headers }).headers
    if (!headers) return { user: null }
    const session = await auth.api.getSession({ headers })
    return { user: session?.user ?? null }
  }),

  // Invalidates current session — protected
  // Better Auth's server-side sign-out: use signOut via the API handler
  logout: requireAuth.handler(async ({ context }) => {
    const headers = (context as { headers: Headers }).headers
    // Revoke the current session server-side
    await auth.api.revokeSession({ headers })
    return { success: true }
  }),
})
```
