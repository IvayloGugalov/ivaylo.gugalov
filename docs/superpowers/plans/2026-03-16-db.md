# Portfolio Site — DB Agent Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the full Drizzle ORM layer — Neon client, all custom table schemas, and the migration runner.

**Architecture:** Feature-sliced schema files (`src/server/*/schema.ts`) re-exported from a central `src/server/db/schema.ts`. Neon serverless driver with Drizzle ORM. Better Auth generates its own schema via CLI — db-agent merges it into the central schema file.

**Tech Stack:** drizzle-orm, @neondatabase/serverless, drizzle-kit, Better Auth CLI

**Prerequisite:** Phase 0 setup complete (deps installed, `.env.local` configured, `drizzle.config.ts` exists).

**Task order matters:** `schema.ts` must exist before `client.ts` is usable. Follow the order below.

### Task 3: comments + reactions schema (with FK constraints)

**Files:**
- Create: `src/server/comments/schema.ts`

- [ ] **Step 1: Write comments and reactions tables**

Note: `userId` references `users.id` from the Better Auth schema. Since Better Auth schema is generated in Task 4, we define the FK as a plain string column here and Drizzle will enforce the constraint via the migration SQL — the `users` table will exist by the time migrations run.

Create `src/server/comments/schema.ts`:

```ts
import {
  pgTable,
  text,
  timestamp,
  unique,
  uuid,
} from 'drizzle-orm/pg-core'

// Forward-reference to Better Auth users table (generated in Task 4)
// FK constraint is expressed via .references() pointing to the table name string
export const comments = pgTable('comments', {
  id: uuid('id').primaryKey().defaultRandom(),
  postSlug: text('post_slug').notNull(),
  // FK to users.id — Better Auth owns the users table
  userId: text('user_id').notNull(),
  content: text('content').notNull(),
  // Self-referencing FK: null = top-level, non-null = reply (max 1 level enforced in router)
  parentId: uuid('parent_id'),
  createdAt: timestamp('created_at').notNull().defaultNow(),
  updatedAt: timestamp('updated_at').notNull().defaultNow(),
})

export const reactions = pgTable(
  'reactions',
  {
    id: uuid('id').primaryKey().defaultRandom(),
    targetId: text('target_id').notNull(),
    targetType: text('target_type', { enum: ['post', 'comment'] }).notNull(),
    // FK to users.id — Better Auth owns the users table
    userId: text('user_id').notNull(),
    emoji: text('emoji').notNull(),
    createdAt: timestamp('created_at').notNull().defaultNow(),
  },
  (t) => [unique().on(t.targetId, t.targetType, t.userId, t.emoji)],
)
```

- [ ] **Step 2: Write schema tests**

Create `src/server/comments/schema.test.ts`:

```ts

    expect(cols.id).toBeDefined()
    expect(cols.postSlug).toBeDefined()
    expect(cols.userId).toBeDefined()
    expect(cols.content).toBeDefined()
    expect(cols.parentId).toBeDefined()
    expect(cols.createdAt).toBeDefined()

    expect(comments.id.primary).toBe(true)
  })

  it('targetType enum contains post and comment', () => {

  })
```

**Files:**

- [ ] **Step 1: Update comments schema with FK references**

Now that `users` is available from the Better Auth schema, add `.references()` to the FK columns. Modify `src/server/comments/schema.ts` — replace the FK comment lines:

```ts
import {
  pgTable,
  text,
  timestamp,
  unique,
  uuid,
} from 'drizzle-orm/pg-core'
import { users } from '#/server/db/schema'

export const comments = pgTable('comments', {
  id: uuid('id').primaryKey().defaultRandom(),
  postSlug: text('post_slug').notNull(),
  userId: text('user_id').notNull().references(() => users.id, { onDelete: 'cascade' }),
  content: text('content').notNull(),
  parentId: uuid('parent_id'),
  createdAt: timestamp('created_at').notNull().defaultNow(),
  updatedAt: timestamp('updated_at').notNull().defaultNow(),
})

export const reactions = pgTable(
  'reactions',
  {
    id: uuid('id').primaryKey().defaultRandom(),
    targetId: text('target_id').notNull(),
    targetType: text('target_type', { enum: ['post', 'comment'] }).notNull(),
    userId: text('user_id').notNull().references(() => users.id, { onDelete: 'cascade' }),
    emoji: text('emoji').notNull(),
    createdAt: timestamp('created_at').notNull().defaultNow(),
  },
  (t) => [unique().on(t.targetId, t.targetType, t.userId, t.emoji)],
)
```

Create `src/server/db/migrate.ts`:

```ts
import { neon } from '@neondatabase/serverless'
import { drizzle } from 'drizzle-orm/neon-http'
import { migrate } from 'drizzle-orm/neon-http/migrator'

async function main() {
  const sql = neon(process.env.DATABASE_URL!)
  const db = drizzle(sql)

  console.log('Running migrations...')
  await migrate(db, { migrationsFolder: './drizzle' })
  console.log('Migrations complete.')
  process.exit(0)
}

main().catch((err) => {
  console.error('Migration failed:', err)
  process.exit(1)
})
```
